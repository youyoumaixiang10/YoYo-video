# 视频制作合同

仅在完整画面脚本获得有效确认、进入 `PRODUCTION_APPROVED` 后使用本合同。

## 成片规格

- 画幅与分辨率：9:16 竖屏，1080 × 1920。
- 帧率：25fps。
- 封装与编码：MP4，H.264 视频，AAC 音频。
- 总时长：20—35 秒。
- 幕数：5—7 幕。

## 文字、可选音乐与旁白边界

- 成片必须在静音状态下看懂核心事件与因果；不得依赖旁白补齐画面缺失的信息。
- 每屏同时显示的字幕、对白或收束文字不超过 3 行，每行约 12 个汉字。超过 3 行时按语义分阶段显示，并为每个阶段截取可读性检查帧。
- 中文文字在后期添加，不由插画模型直接生成。
- 默认不生成旁白音频。真实童言、连续提问和关键转折必须完整写在画面上，例如“下巴是干什么的？肚子是干什么的？头发是干什么的？”要逐句出现，不得压缩为概括句。
- 只有用户明确要求旁白并确认旁白文本后，才可生成配音；此时默认使用妈妈第一人称、自然讲故事、不使用播音腔，并完整朗读关键原话。
- 启用配音时，必须测量音频实际时长，并让时间线片段与音频时长匹配且不重叠。时长不足时优先删减非必要旁白，不得牺牲关键原话。
- 背景音乐同样为可选项；启用时保持低存在感，可使用轻钢琴、木吉他、八音盒或低存在感 Lo-fi。

## 构图、气泡与背景一致性

- 按 `visual-direction.md` 检查文字与人物的视觉分量；大面积留白不能以人物过小、表情动作看不清为代价。
- 对话框必须是自然、不规则、略非对称的手绘气泡，尾巴准确指向说话者；禁止规整药丸形或完美椭圆 UI 气泡。
- 插画缩放、蒙版显现或转场露出底层画布时，从插画边缘取样纸色并匹配根背景，消除偏黄矩形、色差和接缝。
- 红圈、红点、箭头等装饰必须有准确目标和明确叙事作用；无意义、错位或无法解释的标记必须移除。

## 每幕视觉变化

每幕至少包含两次可见的视觉变化。变化从以下绘制状态中选择并按绘制进程推进：

1. 空白背景。
2. 局部线稿：人物脸部与主要动作先出现。
3. 完整线稿：人物与场景线稿完整。
4. 局部上色。
5. 完整低饱和上色。

无法获得真实图层时，可以用蒙版、渐进擦除、局部显现或轻微推近模拟绘制过程；成片不得退化为静态图片轮播。

## 制作验收

### 三阶段代表帧检查

在最终导出前，每一幕至少抽取 3 张带时间戳的代表帧，分别覆盖：该幕初始状态、第一次可见变化完成后、第二次可见变化完成后。逐幕并逐对比较“初始 → 第一次变化后”和“第一次变化后 → 第二次变化后”；只有三个阶段均存在可辨识差异，且两次变化都符合已确认的绘制进程，动态验收才通过。单帧、无时间戳帧或三个近似静态帧不得作为通过证据。

同时用这 3 张帧逐幕核对：角色年龄感、发型、脸型、身材、固定穿搭与人物比例；构图、动作、表情和文字是否符合已确认画面脚本；字幕是否清晰并处于安全区域。

若某幕包含分阶段文字、完整连续提问或多段收束，除三阶段代表帧外，再为每个最终可读状态增加一张关键帧。静音逐幕检查：因果是否完整、每句关键对白是否出现、文字是否溢出、人物是否过小、气泡尾巴是否指对、背景是否有色差接缝、是否存在无意义红点或红圈。

运行 HyperFrames 项目检查时，必须处理全部错误和警告后再导出；目标为 0 error、0 warning，并尽量清除 info。

### 最终媒体属性验证

交付前必须对实际导出的文件核验：文件可读取且确为 MP4；分辨率为 1080 × 1920；帧率为 25fps；视频编码为 H.264；总时长为 20—35 秒；画面可正常播放。若启用了音乐或旁白，还要核验音频编码为 AAC、音画同步，并确认音乐不盖过人声。只有适用属性全部验证通过，才能报告成片路径。
